5. FLOWCHART - KELOLA DONASI/INFAQ
==============================================

```plantuml
@startuml
title SIMASJID - Flowchart Kelola Donasi/Infaq

start
:Admin Login ke SIMASJID;
:Pilih Menu Kelola Donasi;
:Tampilkan Daftar Donasi;

:Pilih Aksi;
switch (Aksi yang dipilih?)
case (Lihat Detail)
  :Tampilkan detail donasi lengkap;
  :Lihat riwayat donatur;
  
case (Update Status)
  :Pilih donasi untuk update;
  :Ubah status donasi;
  if (Konfirmasi status?) then (ya)
    :Simpan status baru;
    :Kirim notifikasi ke donatur;
    note right: Email/SMS konfirmasi\nstatus donasi
  else (tidak)
    :Batalkan update;
  endif
  
case (Export Laporan)
  :Pilih periode dan filter;
  :Generate laporan donasi;
  :Download file Excel/PDF;
  note right: Laporan berisi:\n- Total donasi\n- Detail donatur\n- Grafik periode
  
case (Tambah Manual)
  :Input data donasi manual;
  :Validasi data input;
  if (Data valid?) then (ya)
    :Simpan donasi baru;
  else (tidak)
    :Tampilkan error validation;
  endif
  
case (Filter/Cari)
  :Input kriteria pencarian;
  :Filter berdasarkan periode/status;
  :Tampilkan hasil filter;
endswitch

:Refresh daftar donasi;
:Tampilkan notifikasi sukses;
if (Lakukan aksi lain?) then (ya)
  goto Pilih Aksi
else (tidak)
  stop
endif

@enduml
```

6. FLOWCHART - BOOKING JANJI TEMU (GUEST)
==============================================

```plantuml
@startuml
title SIMASJID - Flowchart Booking Janji Temu (Guest)

start
:Guest akses halaman Janji Temu;
:Tampilkan form multi-step booking;

partition "Step 1: Data Personal" {
  :Input data personal\n(Nama, Email, Telepon);
  :Validasi format data;
  if (Data personal valid?) then (tidak)
    :Tampilkan error validation;
    goto Input data personal
  else (ya)
    note right: Data tersimpan\nsementara di session
  endif
}

partition "Step 2: Pilih Ustadz" {
  :Load daftar ustadz aktif;
  :Tampilkan profile & spesialisasi;
  if (Ustadz dipilih?) then (tidak)
    :Pilih ustadz lain;
    goto Load daftar ustadz aktif
  else (ya)
    :Set ustadz_id;
  endif
}

partition "Step 3: Pilih Jadwal" {
  :Load jadwal ustadz terpilih;
  :Generate available time slots;
  if (Waktu tersedia?) then (tidak)
    :Tampilkan "Slot penuh";
    :Suggest alternatif waktu;
    goto Load jadwal ustadz terpilih
  else (ya)
    :Set tanggal & waktu;
  endif
}

partition "Step 4: Detail Keperluan" {
  :Input keperluan konsultasi;
  :Validasi minimal karakter;
  if (Keperluan valid?) then (tidak)
    :Tampilkan error;
    goto Input keperluan konsultasi
  else (ya)
    note right: Min 20 karakter
  endif
}

partition "Review & Submit" {
  :Tampilkan ringkasan booking;
  :Show detail lengkap;
  if (Konfirmasi submit?) then (tidak)
    :Kembali untuk edit;
    goto Tampilkan form multi-step booking
  else (ya)
    :Validasi final & check conflict;
    if (No conflict?) then (tidak)
      :Tampilkan error conflict;
      :Suggest waktu alternatif;
      goto Load jadwal ustadz terpilih
    else (ya)
      :Simpan janji temu;
      :Set status = "pending";
      :Kirim notifikasi ke admin;
      :Kirim konfirmasi ke guest;
      :Tampilkan success message;
    endif
  endif
}

stop

@enduml
```

9. INSTRUKSI PENGGUNAAN PLANTTEXT
==============================================

**CARA MENGGUNAKAN DI PLANTTEXT.COM:**

1. **Buka** www.planttext.com
2. **Copy** salah satu kode PlantUML dari file ini
3. **Paste** ke text editor di PlantText
4. **Klik** "Generate" atau preview otomatis
5. **Download** sebagai PNG/SVG jika perlu

**TIPS PLANTTEXT:**
- Setiap diagram dimulai dengan `@startuml` dan diakhiri `@enduml`
- Syntax highlighting otomatis tersedia
- Real-time preview membantu debugging
- Support export ke berbagai format
- Bisa share via URL untuk kolaborasi

**KUSTOMISASI WARNA SIMASJID:**
- Primary Green: `#2E7D32` (tema islami)
- Secondary Blue: `#1976D2` (teknologi)
- Light variants untuk background
- Konsisten di semua diagram

**DIAGRAM YANG TERSEDIA:**
1. Use Case Diagram - Fungsionalitas sistem
2. Activity Diagram - Alur login admin
3. Sequence Diagram - Interaksi kelola kas
4. Class Diagram - Struktur kelas & relasi
5. Flowchart Donasi - Proses kelola infaq
6. Flowchart Booking - Alur janji temu guest
7. Database Schema ERD - Struktur database
8. UI Wireframe - Layout admin dashboard

**UNTUK PENGEMBANGAN:**
- Gunakan diagram class untuk coding
- Gunakan ERD untuk database migration
- Gunakan sequence untuk API design
- Gunakan wireframe untuk UI/UX design
